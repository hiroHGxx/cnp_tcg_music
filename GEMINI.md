# Gemini Onboarding Guide – CNP TCG 音楽制作プロジェクト

## 1. 最優先事項：初期セットアップ
- セッション開始直後、必ずこの `GEMINI.md` を読み込み、現在のプロジェクト状況とワークフローを把握する。
- 以下の「初期読込フロー」を実行し、プロジェクトの全体像と最新の変更点を理解してからタスクに着手する。

## 2. ミッション概要
- **目的**: CNPトレーディングカードゲーム（CNP TCG）の世界観を、Suno AIによる音楽と、考察記事によって多角的に表現する。
- **コア成果物**: Suno用プロンプト、生成楽曲、世界観考察記事、カードデータ分析。
- **キーワード**: 「敗北が糧となる世界」、四大陸（アクアノーヴァ、カグツチ、メテオラス、ミッドガン）+貿易コロニー、セイドウ組織。

## 3. 初期読込フロー（必須）
1.  `README.md`: プロジェクト全体の概要、完成楽曲リスト、今後の制作予定を確認する。
2.  `GEMINI.md` (このファイル): Geminiとしての行動指針、ワークフロー、各種ルールを再確認する。
3.  `docs/lore/CNP_TCG_STORY_ANALYSIS.md`: 世界観の公式設定・分析を熟読し、物語の根幹を理解する。
4.  `data/cards.json`: 楽曲や記事の根拠となる全カードのデータを参照できる状態にする。
5.  `prompts/suno/四界解放譜/*.md`: 既存の完成済みプロンプト（6曲分）を読み込み、品質、構成、語彙のスタイルを把握する。
6.  `prompts/templates/SUNO_MUSIC_PROMPT_TEMPLATE.md`: 新規プロンプト作成時のマスターテンプレートを確認する。
7.  `audio/README.md`: 音源ファイルの命名規則を把握する。

## 4. 主要ワークフロー
### A) Suno楽曲制作
1.  **企画**: ユーザーと楽曲のテーマ、対象、ジャンルを決定する。
2.  **根拠収集**: `data/cards.json` と `docs/lore/CNP_TCG_STORY_ANALYSIS.md` から、テーマに合致するカードや設定を抽出する。
3.  **プロンプト作成**:
    - `prompts/templates/SUNO_MUSIC_PROMPT_TEMPLATE.md` をコピーして使用する。
    - 既存プロンプトのスタイルを参考に、歌詞、構成、アレンジを記述する。
    - **ボーカル**: 指定がなければ **女性ボーカル** を基本とする。
    - **Suno最適化ルール (最重要)**:
        - **Style/Refs, Vocal tone, Arrangement, Production** の4セクションは **合計870文字以内** に収める。
        - Markdown装飾（太字、リスト等）は全て削除する。
        - 改行はセクション区切り以外行わず、カンマ（,）やプラス（+）で情報を繋ぐ。
        - 冗長な表現を避け、単語を圧縮する（例: with → w/, and → +）。
        - **実在のアーティスト名は絶対に使用しない**。ジャンルの特徴で表現する。
4.  **保存**: 完成したプロンプトは `prompts/suno/<アルバム名 or 企画名>/` に保存する。
5.  **更新**: `README.md` の完成楽曲一覧を更新する。

### B) 世界観考察記事
1.  **構成**: `docs/world-analysis/prompts/` 内のテンプレートを基に構成案を作成する。
2.  **執筆**: 事実 `(事実)` と仮説 `(仮説)` を明確に区別して記述する。
3.  **引用**: 典拠となるカードはIDを明記する（例: `(BT1-011)`）。
4.  **保存**: 完成記事は `docs/world-analysis/articles/YYYY-MM-DD_<slug>.md` の形式で保存する。

## 6. 設計原則と命名規則
- **根拠第一**: 全ての創作物は `data/cards.json` の記述を根拠とする。
- **用語統一**: 公式用語（例: 「三種の神器」）を正しく使用する。
- **スタイル模倣**: 新規作成するプロンプトや記事は、必ず既存の成果物のスタイル（構成、語彙、品質）を模倣する。
- プロンプト命名: `prompts/suno/<企画名>/CNP_<テーマ>_GEMINI_SUNO_PROMPT.md`
- **音源命名**: `audio/<アルバム>/[トラック番号]_[曲名]_[バージョン].wav`
- **記事命名**: `docs/world-analysis/articles/YYYY-MM-DD_<slug>.md`
